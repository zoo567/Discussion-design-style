# 咖乐 UI 风格探索网页：移动端响应式与交互优化任务清单

目标：在不破坏桌面端现有视觉结构的前提下，优化手机端与窄屏端的浏览效率、信息层级和交互习惯。

## 任务 1：锁定桌面端结构，移动端单独优化

- 保留桌面端顶部栏、页面切换、类型筛选、参考预览、风格解析的现有结构。
- 所有主要改动优先写入移动端媒体查询，避免桌面端出现二次偏移。

## 任务 2：优化移动端顶部区信息层级

- 移动端隐藏辅助装饰信息与展开式数据区，减少顶部占屏。
- 将 `@zhang`、页面说明、参考数量、来源说明合并到一个问号说明浮层。
- 顶部保留页面切换与必要筛选，确保用户先看到内容，而不是被导航占据半屏。

## 任务 3：重定义移动端参考预览交互

- 参考预览保留“快速扫图”的目标，手机端使用更紧凑的单列/双列规则。
- 手机端点击卡片进入对应风格解析，不再把整张卡片点击定义为全屏查看。
- 仅保留明确的放大按钮用于图片大图查看，减少误触。

## 任务 4：重定义移动端风格解析交互

- 桌面端保留左侧序号导航与上下切换。
- 手机端隐藏桌面式上下箭头与纵向序号栏，改为更符合触屏的横向序号条/自然滚动阅读。
- 风格解析中参考图与分析文字改为上下排列，避免图片与分析区重叠。

## 任务 5：修正响应式视觉关系

- 移动端减少顶部区、筛选区、内容区之间的垂直空隙。
- 参考预览卡片、标签、标题字号按手机端重新收敛。
- 风格解析的图片容器、文字区、标签区保持清晰层级。

## 任务 6：验证交互与响应式

- 检查桌面端、iPad 宽度、手机宽度三种状态。
- 检查参考预览卡片点击、放大按钮、类型筛选、页面切换、风格解析序号跳转。
- 检查键盘上下键与桌面端上下箭头不受移动端规则影响。
